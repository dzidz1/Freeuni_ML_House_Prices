# House Prices - Advanced Regression Techniques

კონკურსის აღწერა: კონკურსის მიზანია ისეთი მოდელის ტრენინგი, რომელიც მოცემული feature-ების საშუალებით შეძლებს სახლების ფასების პროგნოზირებას.

ჩემი მიდგომა: მონაცემები შევისწავლე train.csv-ისა და data_description.txt ფაილების საშუალებით და მათზე დაყრდნობით მივყევი cleaning, feature engineering, feature selection ეტაპებს.

რეპოზიტორიის სტრუქტურა: model_experiment.ipynb ფაილი შეიცავს სრულ ექსპერიმენტულ და ანალიტიკურ პროცესებსა და MLflow-ზე ლოგირების ნაწილს, სექციები კი გამოყოფილია შესაბამისი Heading-ებით. model_inference.ipynb ფაილში Model Registry-დან ვტვირთავ საუკეთესო მოდელს და მისი გამოყენებით ვაგენერირებ submissions.csv ფაილს Kaggle-ზე Score-ის შესამოწმებლად. README.md-ში შეძლებთ ნახოთ ჩემს მიერ გამოყენებული მიდგომები და მათი შედეგები.

Feature Engineering: თავდაპირველად ზუსტად არ ვიცოდი როგორ დამეწყო პროცესი და გადავწყვიტე ყველა feature-ის დამოკიდებულებას დავკვირვებოდი SalePrice-თან მიმართებით და თუ დავიჭერდი outlier-ებს, manually ვდროპავდი. ერთი დღე სრულად ამაში გავხარჯე, თუმცა შემდეგ მივხვდი, რომ ამგვარ მიდგომას შეიძლებოდა გამოეწვია biased შედეგები, თუ შემთხვევით წავშლიდი valid data-საც. ამის შემდეგ გავაანალიზე, რომ უფრო მნიშვნელოვანი ეტაპები მქონდა გასავლელი, ვიდრე outlier-ების დადროპვა და ეს საქმე ცოტა ხანს გვერდზე გადავდე. დავიწყე Missing Value-ებზე დაკვირვება ვიზუალიზაციის ტექნიკების საშუალებით. დავდროპე ისეთი სვეტები, რომლებიც არ ატარებდნენ მნიშვნელოვან ინფორმაციას მოდელისთვის ან NaN-ების წილი აჭარბებდა 80%-ს. რადგან ყველა კატეგორიული ცვლადი მსგავსი შინაარსის არ იყო, გამოვიყენე ორგვარი მიდგომა: Ordinal Encoding, იმ სვეტებისთვის, რომელთა მნიშვნელობებს შორის არსებობდა ლოგიკური იერარქია და One-Hot Encoding მათთვის, რომელთანაც მსგავსი კავშირები არ გვხვდებოდა. კოდში ეს პროცესი ავტომატიზებულია ColumnTransformer-ით, რომელიც მოიცავს Imputation (სანამ რიცხვითში გადავიყვან პოულობს გამოტოვებულ მონაცემებს და ავსებს ყველაზე ხშირად განმეორაბადი მნიშვნელობით), Transformation (OrdinalEncoder და OneHotEncoder ასრულებენ კატეგორიულების რიცხვითში კონვერტირებას) და Handling Unknowns ეტაპებს. გარდა ამისა, დავამატე რამდენიმე feature, მაგალითად: HouseAge, HouseRemodelAge, TotalSF, TotalArea და სხვა. Data Cleaning-ისთვის მაინც დავუბრუნდი outlier-ებს, მაგრამ ისევ უშედეგოდ. ვისწავლე IQR და Z-score სტატისტიკური მეთოდები და გამოვიყენე კიდეც. მათი გამოყენებით მოდელი კარგ შედეგს მაძლევდა train-ზე, თუმცა Kaggle-ზე საშინლად ცუდი რეზულტატი დადო, რაც overfitting-ის მაგალითია. მსგავსად დამემართა target encoding-ისა და Quality mapping-ის გამოყენებისასაც, ამიტომ გადავწყვიტე საერთოდ არ გამომეყენებინა ეს ტექნიკები. მოდელის შესაფასებლად ვიყენებდი Root-Mean-Squared-Error (RMSE)-ს პროგნოზირებული ფასებისა და რეალური ფასების ლოგარითმებს შორის, რადგან Kaggle-ზე სწორედ ამ მეტრიკით ხდებოდა submission-ის ევალუაცია.

Feature Selection: გამოვიყენე Correlation Filter, რომლის threshold-ად ავიღე 0.9 და ასევე RFE, რომლის საშუალებითაც საუკეთესო 60 feature შეირჩა.

Training: საწყის ეტაპზე გამოვიყენე Linear Regression, რომელმაც ვალიდაციაზე მაღალი სიზუსტე (RMSE: 0.113) აჩვენა, თუმცა Kaggle-ის ტესტ სეტზე ვერ განაზოგადა (Score: 0.313) და გამოავლინა კლასიკური overfitting. ამის შემდეგ ვცადე Decision Trees, სადაც გამოიკვეთა, რომ ღრმა ხეები ზედმეტად ერგებოდა საწვრთნელ მონაცემებს, ხოლო ნაკლები სიღრმის მქონე ხეები ვერ ითვისებდნენ საჭირო კომპლექსურობას, ვლინდებოდა underfitting. შედარებით სტაბილური შედეგები აჩვენა Random Forest-მა და XGBoost-მა. მოდელების ეფექტურობის მაქსიმალურად გაზრდის მიზნით გამოვიყენე Optuna-ს framework ჰიპერპარამეტრების ოპტიმიზაციისთვის. იმ მოდელებისთვის, რომლებმაც საუკეთესო შედეგები მომცეს, ალგორითმი ავტომატურად ეძებდა ოპტიმალურ მნიშვნელობებს ისეთი პარამეტრებისთვის, როგორებიცაა learning rate, tree depth და regularization coefficient. მნიშვნელოვანია, რომ ოპტიმიზაციის პროცესი ეყრდნობოდა 5-Fold Cross-Validation მიდგომას, რომელიც მიმცირებდა შემთხვევითი წარმატების ალბათობას და ზრდიდა მოდელის მდგრადობას. საბოლოო მოდელად შევარჩიე Weighted Voting Ensemble, რომელიც აერთიანებს XGBoost, LightGBM, Ridge და Lasso მოდელებს. ამგვარად, ინდივიდუალური მოდელების შეცდომების კომპენსირება გახდა შესაძლებელი. პრიორიტეტი მივანიჭე ძლიერ ბუსტინგ ალგორითმებს, ხოლო Regularized Linear მოდელების საშუალებით გავაუმჯობესე პროგნოზის სტაბილურობა. სწორედ ამ კომბინაციამ მომცა საუკეთესო ბალანსი ვალიდაციის RMSE-სა (0.13487) და Kaggle-ის რეალურ ქულას (0.13641) შორის, რაც ადასტურებს, რომ მოდელი არ არის overfitted და ეფექტურად მუშაობს სრულიად უცნობ მონაცემებზე.
